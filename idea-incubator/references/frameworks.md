# Frameworks crosswalk & optional scoring

This file grounds the interview in proven idea-evaluation frameworks so the
questions aren't invented. Read it when you want sharper follow-ups or when the
user wants a numeric score. Don't dump it into the conversation.

## Where each phase comes from

| Phase | Borrowed from |
|---|---|
| Hero & Pain | Value Proposition Canvas (Strategyzer) — customer jobs/pains/gains; *The Mom Test* (Rob Fitzpatrick) — talk about their life, not your idea |
| Value Proposition | VPC pain relievers / gain creators; "before → after" bridge (Eugene Schwartz / Russ Henneberry) |
| Goal | Start with the win; Amazon "Working Backwards" PR-FAQ — write the press release first |
| Framework (unit of work) | Lean / repeatable production unit; the "smallest shippable thing" |
| Workflow & gates | Lean Startup build-measure-learn loops; stage-gate decision points |
| Viability tests | Sequoia pitch dimensions; YC's "make something people want"; unit economics (LTV vs. CAC) |
| Paths & expectations | Venture outcome distribution (most die, few are base hits, rare unicorns) |
| Point of viability / kill | Pre-mortem (Gary Klein) + pre-committed kill criteria |

## The Mom Test rules (use these to detect bad evidence)

Good demand evidence is **past behavior and commitments**, not opinions about the future.
- Talk about *their life and problems*, not your idea.
- "Would you buy this?" is worthless. "What do you use today / what did you last pay for / show me how you do it now" is gold.
- Compliments, "I would totally use that," and hypotheticals = no signal. Treat them as unvalidated.
- Real signal = they've already hacked a solution, spent money/time, or will commit something now (pre-order, intro, deposit, calendar time).

## Optional viability scorecard (only if the user wants a number)

Score each 1–5 (1 = serious problem, 5 = clear strength). Weight to taste.

| Dimension | Question | Score |
|---|---|---|
| Demand | Do real people pull for this, with evidence (not opinions)? | _ |
| Feasibility | Can you deliver it with reachable resources? | _ |
| Economics | Does value-captured-per-unit beat cost-to-make-and-acquire? | _ |
| Differentiation | Is there a reason it won't be copied/commoditized? | _ |
| Timing & distribution | Why now, and a real path for people to find it? | _ |
| Founder fit | Can you stand the repetitive workflow for years? | _ |

Reading the total (out of 30) is less useful than the **lowest score**: one 1–2 is
a flag to fix before proceeding; two or more is usually a reshape-or-shelve signal.
Always name the weakest dimension and the cheapest test to move it up a point.

## Common failure patterns to watch for

- **Solution in search of a problem** — they're attached to the build, not the hero.
- **Vitamin, not painkiller** — nice-to-have; people won't change behavior.
- **Tiny or fake market** — "everyone" is nobody.
- **Right idea, no distribution** — can build it, can't get it found.
- **Economics that never close** — cost to make/acquire always exceeds value captured.
- **Founder can't stand the workflow** — burns out before the unit count that wins.
